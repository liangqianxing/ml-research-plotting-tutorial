# 投稿检查表

在提交论文或共享最终结果之前，逐项检查图片和生成过程。

## 科学内容

- [ ] 每张图支持一个明确论点；
- [ ] 数据划分、指标方向和样本数量明确；
- [ ] 不确定性类型和重复次数写在图注中；
- [ ] 比较模型使用相同实验协议；
- [ ] 测试集没有参与模型选择或早停；
- [ ] 坐标范围没有夸大差异；
- [ ] 类别顺序具有业务或统计意义。

## 视觉表达

- [ ] 同一模型在所有图片中保持相同颜色与标记；
- [ ] 灰度和常见色觉差异下仍可区分；
- [ ] 正文尺寸下字体、线宽和标记清晰；
- [ ] 图例没有遮挡数据；
- [ ] 面板标签顺序与正文引用一致；
- [ ] 图片不依赖标题重复正文信息。

## 文件交付

- [ ] 线图优先保存为 PDF 或 SVG；
- [ ] 位图满足期刊分辨率要求；
- [ ] 字体已经嵌入；
- [ ] 文件名稳定且不包含 `final-final-2`；
- [ ] 原始数据和绘图脚本已归档；
- [ ] 依赖版本和随机种子已记录；
- [ ] 从干净环境可以重新生成全部图片。

## 推荐目录

```text
project/
├── data/
│   ├── raw/
│   └── processed/
├── results/
│   └── metrics.csv
├── scripts/
│   ├── train.py
│   └── plot.py
├── figures/
│   ├── figure-1.pdf
│   └── figure-1.png
├── requirements-lock.txt
└── README.md
```

::: tip 最后一次检查应脱离开发环境
使用论文 PDF 或投稿系统生成的最终预览检查图片，而不只是在绘图窗口中查看。
:::
