# 09 投稿检查表

在提交论文或共享最终结果之前，逐项检查图片和生成过程。

::: info NOTE
**本章导学**

- 检查每张图是否支持一个明确且可验证的论点
- 检查颜色、字号、图例和坐标范围是否容易误读
- 检查文件格式、字体嵌入和分辨率是否符合要求
- 从干净环境重新生成并核对最终论文预览
:::

## 科学内容

- [ ] 每张图支持一个明确论点；
- [ ] 数据划分、指标方向和样本数量明确；
- [ ] 不确定性类型和重复次数写在图注中；
- [ ] 比较模型使用相同实验协议；
- [ ] 测试集没有参与模型选择或早停；
- [ ] 坐标范围没有夸大差异；
- [ ] 类别顺序具有业务或统计意义。

## 视觉表达

- [ ] 同一模型在所有图片中保持相同颜色与标记；
- [ ] 灰度和常见色觉差异下仍可区分；
- [ ] 正文尺寸下字体、线宽和标记清晰；
- [ ] 图例没有遮挡数据；
- [ ] 面板标签顺序与正文引用一致；
- [ ] 图片不依赖标题重复正文信息。

## 方法示意图

- [ ] 总览图可以在十秒内说明输入、核心贡献和输出；
- [ ] 主流程只有一个明确的阅读方向；
- [ ] 数据、模块、状态与输出使用稳定的形状语义；
- [ ] 实线、虚线、反馈箭头和分组边界含义一致；
- [ ] 只有一个第一视觉焦点，核心贡献没有被装饰抢走注意力；
- [ ] 同类模块等宽、等高、对齐且间距一致；
- [ ] 图中术语、符号、尺寸顺序与正文完全一致；
- [ ] 训练阶段、推理阶段与可选路径已经清楚区分；
- [ ] PPT 源文件与最终 PDF/SVG 一同归档。

## 外部论文案例

- [ ] 案例确实来自 NeurIPS、ICML 或 ICLR 的 Oral 论文，而不是只凭论文知名度判断；
- [ ] 已链接会议 Oral 页面或官方 Oral 列表；
- [ ] 已标明论文题目、会议年份、Figure 编号和原始图注；
- [ ] 图片只做必要裁切，没有改色、重绘或省略关键上下文；
- [ ] 已声明图片用于教学评论，著作权归原作者及出版方；
- [ ] 教程自制图与论文原图有明确区分，不会让读者误认来源。

## 文件交付

- [ ] 线图优先保存为 PDF 或 SVG；
- [ ] 位图满足期刊分辨率要求；
- [ ] 字体已经嵌入；
- [ ] 文件名稳定且不包含 `final-final-2`；
- [ ] 原始数据和绘图脚本已归档；
- [ ] 依赖版本和随机种子已记录；
- [ ] 从干净环境可以重新生成全部图片。
- [ ] PPT 示意图已在论文最终栏宽检查，而不是只在编辑界面放大查看；
- [ ] 示意图中的连接符、文字和基础形状保持矢量。

## 推荐目录

```text

├── data/
│   ├── raw/

├── results/
│   └── metrics.csv

├── figures/

```

::: tip 最后一次检查应脱离开发环境
使用论文 PDF 或投稿系统生成的最终预览检查图片，而不只是在绘图窗口中查看。
:::
