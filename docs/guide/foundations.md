# 绘图原则

科研绘图的目标是让读者准确理解证据，而不是让图表看起来更复杂。图型、坐标、颜色和标注都应当从研究问题出发。

## 先写出图表论点

绘图前先完成一句话：

> 这张图需要证明 ______，读者将通过比较 ______ 得出结论。

例如：“改进模型在三个数据集上都提高了 Macro F1，且提升幅度大于不同随机种子产生的波动。”这句话决定了图中至少需要模型、数据集、均值与重复实验信息。

## 变量到视觉通道的映射

| 数据角色 | 优先视觉通道 | 常见用途 |
| --- | --- | --- |
| 定量值 | 位置、长度 | 分数、时间、参数量 |
| 有序类别 | 位置、明度 | Epoch、数据规模、噪声水平 |
| 无序类别 | 颜色、形状、线型 | 模型、数据集、实验组 |
| 不确定性 | 区间、分布、原始点 | 标准差、置信区间、重复实验 |

位置通常比面积和颜色深浅更容易精确比较。需要读者比较数值时，优先使用共享坐标轴上的点或线。

## 均值不是完整结果

机器学习结果受到数据划分、初始化、采样顺序和硬件非确定性的影响。只画均值会隐藏实验稳定性。

```python
summary = (
    results.groupby(["model", "dataset"])["macro_f1"]
    .agg(mean="mean", std="std", count="count")
    .reset_index()
)
```

当重复次数较少时，原始点通常比箱线图更诚实；当样本较多时，可以使用小提琴图、ECDF 或箱线图概括分布。

::: warning 误差棒必须说明含义
图注中明确写出误差棒代表标准差、标准误还是置信区间，并说明重复实验次数。
:::

## 颜色不能单独承担分类

颜色应与线型、点型或直接标签配合。建议控制主色数量，并保持同一模型在所有图中的颜色一致。

```python
MODEL_STYLE = {
    "Proposed": {"color": "#DF4B36", "marker": "o", "linestyle": "-"},
    "Baseline": {"color": "#147D73", "marker": "s", "linestyle": "--"},
    "Compact": {"color": "#B47B17", "marker": "^", "linestyle": ":"},
}
```

## 坐标轴从哪里开始

柱状图通过长度编码数值，通常应从零开始。折线图和点图通过位置编码数值，可以截取合理区间，但必须清楚显示刻度，避免制造夸张差异。

## 图注承担实验上下文

一条完整图注至少说明：

1. 图中比较了什么；
2. 数据集、划分和评价指标；
3. 汇总统计与误差定义；
4. 重复次数或样本数；
5. 越大越好还是越小越好。
