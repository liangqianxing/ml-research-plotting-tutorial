# 14 视觉层级与精修：让图专业而克制

精修不是给图加更多效果，而是让重要信息更快被看见，让次要信息不打扰阅读。很多“看起来不高级”的示意图，本质上是所有元素都在用相同音量说话。

::: info NOTE
**本章导学**

- 用位置、尺度、留白、色彩和描边建立三层视觉权重
- 统一字体、对齐、间距与线条，让图呈现稳定秩序
- 使用缩略图、模糊、灰度和最终尺寸进行视觉检查
- 识别常见的“伪精致”效果并完成最后一轮删减
:::

## 建立三层视觉权重

一张论文示意图通常只需要三层：

| 层级 | 内容 | 视觉处理 |
| --- | --- | --- |
| 第一层 | 核心贡献、主结论 | 主色、较粗边框、更大留白或略大尺寸 |
| 第二层 | 主流程、阶段和必要模块 | 常规深色文字、标准描边 |
| 第三层 | 尺寸、条件、辅助损失、解释 | 较小字号、中性灰、细线或虚线 |

第一层只能有一个主要焦点。多个创新点可以先归入同一个“核心方法”区域，再在区域内部建立次序。

## 注意力来自反差

“引人入目”并不等于“把重点涂得最亮”。以下反差都能建立焦点：

- 密集区域旁边的一块留白；
- 一组灰色模块中的一个主色描边；
- 一排同尺寸对象中的一个略大对象；
- 一条连续主路径上的局部放大；
- 大量直线结构中的一个明确汇合点。

每次只选择一到两种反差。过多强调手段叠加，会让图像海报而不是论文。

## 留白也是信息

留白用来说明关系远近和层级边界。可以建立三档间距：

```text
模块内部边距      1x
同一阶段对象间距  2x
不同阶段之间间距  4x
```

例如基础单位为 `0.25 cm` 时，对应 `0.25 / 0.50 / 1.00 cm`。不必机械遵守具体数字，但整张图必须能看出这三种关系。

边缘留白尤其重要。对象贴近画布边缘会产生裁切感，也会让图放入论文版式后显得拥挤。

## 字体只保留必要层级

科研示意图不需要多种字体。与论文正文或图表使用同一无衬线字体最稳妥。

- 阶段标题：全图最大，粗体；
- 模块名称：常规字重，保持一致字号；
- 注释与尺寸：小一档，使用中性灰；
- 数学符号：与正文记号一致，必要时从公式编辑器导入矢量对象；
- 英文标签使用正常大小写，不用长句全大写。

不要通过压缩字宽、减小字间距或把字号降到 6 pt 以下来塞入文字。先缩短标签，再扩大模块或删减信息。

::: tip 中文与英文混排
中文使用系统无衬线字体，英文和数字使用与论文图表一致的字体。避免在同一个短标签中出现明显不同的字重与基线。
:::

## 对齐要看视觉边界

PPT 的几何对齐是起点，不是终点。圆形、三角形和带箭头的对象即使几何中心对齐，也可能看起来偏移。完成自动对齐后，以正常缩放做一次视觉微调。

优先检查：

- 同一行模块的顶部和底部是否形成连续基线；
- 模块文字是否垂直居中，左右内边距是否相等；
- 箭头是否落在模块视觉中心，而不是文字中心；
- 面板标题是否与内部内容左边缘对齐；
- 圆形和菱形是否需要 1–2 px 的光学校正。

## 减少边框、阴影和容器

边框用于定义边界，阴影用于表达层叠。二维方法流程通常没有层叠关系，因此不需要给每个模块加阴影。

可以按顺序删减：

1. 删除所有阴影和渐变；
2. 把普通模块边框降为浅灰；
3. 删除只起装饰作用的外层大框；
4. 保留核心模块的主色边框；
5. 只在确实存在覆盖、卡片或浮层关系时恢复轻微阴影。

页面中如果出现“卡片套卡片”，通常意味着容器层级过多。用留白和阶段标题代替部分外框。

## 做好局部放大

局部放大适合解释一个核心模块，但必须让来源清楚：

- 原位置使用一个明确矩形或圆形标出；
- 放大面板与来源使用同色细线连接；
- 放大面板不重复绘制整条主流程；
- 放大后的比例和方向保持一致；
- 如果局部结构仍然复杂，拆成独立子图 `(b)`。

不要用“爆炸形”“放射线”或厚重投影制造强调，这些效果会占用注意力，却不增加解释能力。

## 四种低成本检查

### 缩略图检查

把整张图缩到页面宽度的 25%。此时仍应看到输入、核心贡献和输出。如果只能看到许多同样大小的框，层级还不够明确。

### 模糊检查

眯眼或给截图增加轻微模糊。文字会消失，只剩色块和结构。检查最大的视觉重量是否落在论文贡献上，而不是某个装饰背景或输入样例上。

### 灰度检查

转成灰度后，主路径、类别和焦点仍应通过明度、形状、线型或标签区分。不能区分时，不要只继续换颜色，应加入冗余编码。

### 最终尺寸检查

把 PDF 放入真实论文模板，在 100% 页面缩放下阅读。检查最小文字、最细线条、面板标签和图注，而不是只在 PPT 编辑界面中放大观看。

## 用三张 Oral 论文图做缩略图审计

把[论文总览图](./overview.md)中的 CLIP、BYOL 和 Glow-TTS 原图都缩到约 25% 观察，可以看到三种不同的“剩余信号”：

| 原图 | 缩小后首先剩下什么 | 说明 |
| --- | --- | --- |
| CLIP Figure 1 | 三个编号区域、紫绿两种模态、左右故事线 | 阶段标题和模态颜色共同维持叙事 |
| BYOL Figure 2 | 上下两条色带、四个阶段列、末端汇合 | 重复网格比单个公式更先被感知 |
| Glow-TTS Figure 1 | 左右镜像骨架、中间两个大机制区 | 对称布局使训练/推理差异更容易定位 |

这项检查不要求缩小后读清每个符号。真正要问的是：文字消失以后，画面结构是否仍然对应论文论点。如果自己的总览图缩小后只剩一团均匀的框和线，就先扩大阶段间距、减少容器或重排主路径，而不是给所有模块增加更亮的颜色。

## 一轮完整精修的顺序

1. **删内容**：移除不支持核心论点的模块、文字和图标；
2. **理结构**：固定阅读方向，减少交叉线和重复层级；
3. **统一几何**：等宽、等高、对齐、分布、内边距；
4. **统一文字**：缩短标签，收敛到三档字号；
5. **统一线条**：只保留普通、核心、辅助三种样式；
6. **分配颜色**：中性底色、一个主色、一个小面积强调色；
7. **导出检查**：最终尺寸、灰度、另一台设备、论文整页预览。

顺序非常重要。结构仍在变化时就精修颜色和阴影，会反复返工，也容易因为舍不得已经完成的细节而保留多余模块。

## 终稿检查表

### 信息

- [ ] 图能用一句话说明任务；
- [ ] 读者能在十秒内指出论文贡献；
- [ ] 主路径只有一个明确方向；
- [ ] 图中术语与正文完全一致；
- [ ] 训练、推理与可选路径已经区分。

### 视觉

- [ ] 只有一个第一视觉焦点；
- [ ] 同类对象使用相同形状、颜色和线宽；
- [ ] 模块之间没有不必要的交叉线；
- [ ] 三档间距和三档文字层级清晰；
- [ ] 灰度下仍能理解类别和主次；
- [ ] 没有装饰性渐变、发光或无意义阴影。

### 交付

- [ ] 已在论文最终栏宽检查；
- [ ] PDF 或 SVG 中的文字和线条保持矢量；
- [ ] 字体已经嵌入或正确转换；
- [ ] 图片边界没有裁掉箭头、标签和面板编号；
- [ ] PPT 源文件与最终导出文件一同归档。

至此，数据图表与手绘示意图可以共用同一套视觉系统。回到[投稿检查表](../reference/checklist.md)，对整篇论文的图片进行统一检查。
