# 11 科研配色：先分配角色，再选择颜色

好看的科研图通常没有很多颜色。它们只是让少数颜色承担了明确任务：建立阅读层级、保持方法身份、提示异常，或者把注意力引向核心贡献。

::: info NOTE
**本章导学**

- 用“角色色”替代“每个框一个颜色”
- 从一个主色构建完整色板，并控制面积、饱和度与对比度
- 选择适合论文、色觉差异与灰度打印的安全配色
- 通过交互示例观察同一图形结构在不同色板下的变化
:::

## 颜色的六种角色

先给颜色命名，再给颜色取值。一个实用的科研图色板通常包含六个角色。

| 角色 | 负责什么 | 推荐使用方式 |
| --- | --- | --- |
| `ink` | 标题、正文、关键轮廓 | 接近黑色，但不要使用纯黑填满大面积 |
| `muted` | 次要文字、辅助线、普通箭头 | 中性灰，必须与背景保持可读对比 |
| `primary` | 论文提出的方法或主路径 | 全文固定，不随页面改变 |
| `secondary` | 对照方法、辅助分支或第二模态 | 与主色有明显色相差，但饱和度相近 |
| `accent` | 新增模块、异常、警告或关键数值 | 面积应很小，只出现于真正需要停留的位置 |
| `surface` | 阶段背景、分组区域 | 主色或中性色的极浅版本 |

如果某个颜色无法对应一个稳定角色，它大概率只是装饰。删除后图通常会更专业。

## 控制颜色面积

科研示意图可以使用比 `60–30–10` 更克制的比例：

- **70%–85% 中性色**：白色、浅灰、深灰文字；
- **10%–20% 主色**：核心模块、关键路径、面板标签；
- **不超过 5% 强调色**：创新标记、异常或最重要结果。

颜色面积越大，饱和度越要降低。大面积背景使用 5%–12% 的浅色填充；高饱和色只适合小面积标签、点或细描边。

::: warning 强调色只能有一个解释
如果橙色有时表示“新模块”，有时表示“性能下降”，读者就无法建立稳定映射。整篇论文中同一种颜色只承担一种主要语义。
:::

## 从 Oral 论文原图反推颜色角色

### BYOL：两种浅色就足以区分双分支

![BYOL 论文 Figure 2，使用极浅蓝紫和浅粉区分 online 与 target 分支](/paper-figures/byol-architecture-figure-2.png)

<div class="paper-caption">
  <strong>NeurIPS 2020 Oral · BYOL Figure 2</strong>
  <p>原图来自 <a href="https://proceedings.neurips.cc/paper/2020/hash/f3ada80d5c4ee70142b17b8192b2958e-Abstract.html"><em>Bootstrap Your Own Latent</em></a>。图片仅从论文 PDF 裁切，未重绘或改色。</p>
</div>

BYOL 没有把 encoder、projector、predictor 分别染成三种颜色，而是让颜色对应**分支身份**。按原图像素近似采样，online 背景约为 `#EBEBFF`，target 背景约为 `#FFEBEB`；两者都接近白色，因此大面积铺设也不会压过黑色箭头和数学符号。

这张图即使去掉颜色仍能读懂，因为它还有三层冗余编码：online 在上、target 在下；分支名称直接写在右侧；stop-gradient 和 loss 使用不同线型。颜色只是降低搜索成本，不是唯一证据。

### CLIP：颜色绑定模态，不绑定步骤

CLIP Figure 1 中，文本编码器与文本向量始终使用浅紫，图像编码器与图像向量始终使用浅绿，选中的相似度格使用浅蓝。三个阶段虽然分散在不同位置，但读者看到紫色就知道是 text，看到绿色就知道是 image。

这种做法比“步骤 1 蓝、步骤 2 绿、步骤 3 紫”更有信息价值。步骤已经由编号和位置表达，颜色应该留给跨面板保持不变的语义。可以把它整理成自己的颜色词典：

| 语义 | CLIP 原图的处理 | PPT 中的复用原则 |
| --- | --- | --- |
| 文本模态 | 浅紫填充、紫色描边 | 所有文本对象共用同一色族 |
| 图像模态 | 浅绿填充、绿色描边 | 图像输入、编码器、向量保持一致 |
| 正确匹配 | 极浅蓝单元格 | 只标选中项，不填满整个矩阵 |
| 普通关系 | 黑色或灰色细线 | 不为每条箭头增加新颜色 |

::: tip 先用一句话定义颜色
不要写“紫色很好看”，要写“紫色表示文本模态”。只有能写成一句稳定定义的颜色，才应该进入论文图。
:::

| 冷静技术 | `#1F2937` | `#3569D4` | `#168C80` | `#DD6047` | `#EEF2F7` |
| 自然医学 | `#20332D` | `#287B69` | `#63779A` | `#C5544D` | `#EDF3F0` |
| 高对比安全 | `#202124` | `#0072B2` | `#009E73` | `#D55E00` | `#F0F2F4` |

“高对比安全”以蓝、绿、橙建立类别差异，适合需要兼顾常见色觉差异的图。但色觉安全不是只选一套色板，还要用形状、线型或直接标签提供冗余编码。

## 从一个主色扩展色板

已经有学校、实验室或项目品牌色时，不要继续随意采样五个颜色。可以按以下顺序扩展：

1. 选择一个中等明度、不过度饱和的主色；
2. 为辅助分支选择相距较远的色相，同时让两者明度接近；
3. 为强调色选择暖色，并显著限制使用面积；
4. 从主色生成 5%–12% 浅色填充，而不是直接降低整个对象透明度；
5. 最后补齐深色文字与中性灰。

透明度会受底色影响，同一个半透明蓝色放在白底和灰底上会变成不同颜色。需要跨软件稳定复现时，直接使用计算后的实色十六进制值。

## 用对比建立层级

引人注意不等于使用更艳的颜色。视觉权重来自多种信号叠加：

- 更深的文字；
- 更粗的描边；
- 更高的色相对比；
- 更大的留白；
- 更居中的位置；
- 更少的邻近元素。

核心模块同时使用其中两项通常已经足够。例如“主色描边 + 更大留白”。如果再叠加高饱和填充、阴影和超大字号，焦点会变成噪声。

## 文字与背景对比

小字号文字优先使用深色文字配浅色背景。不要在浅蓝、浅黄或浅灰背景上使用同色系浅文字。网页和电子论文可参考 WCAG：普通文字对比度尽量达到 `4.5:1`，大号文字至少 `3:1`。

在 PPT 中快速检查：

1. 把画布缩到最终论文宽度；
2. 暂时将屏幕亮度调低；
3. 导出 PDF，在另一台设备打开；
4. 将整页转成灰度，确认主路径仍然可追踪。

## 不要用彩虹色表示连续量

连续量使用明度单调变化的色带，如 Viridis、Cividis；有明确中点的正负变化使用发散色带，并让中点对应中性色。不要因为“更鲜艳”而使用彩虹色，它会制造不存在的边界，也很难判断数值顺序。

| 数据类型 | 推荐色彩策略 | 示例 |
| --- | --- | --- |
| 无序类别 | 彼此可区分的离散色 | 模型 A、B、C |
| 有序类别 | 单一色相逐渐变深 | 数据规模、训练轮数 |
| 连续数值 | 明度单调的连续色带 | 注意力权重、密度 |
| 有中心的差异 | 两端色相 + 中性中心 | 相对基线的提升与下降 |

## 一篇论文只维护一份颜色词典

把角色色放进项目配置，而不是散落在 PPT、Python 和 LaTeX 文件中。

```text
Primary method   #3569D4
Baseline         #667085
Auxiliary branch #168C80
Warning / novel  #DD6047
Panel surface    #EEF2F7
Text             #1F2937
```

图表中的 Proposed、总览图中的核心模块、表格中需要强调的方法名，都应使用同一个 `primary`。这会让整篇论文形成视觉连续性。

## 常见失败模式

- **每个模块一种颜色**：颜色退化成装饰，类别关系反而不清楚；
- **大面积高饱和填充**：屏幕上抢眼，打印和长时间阅读都很疲劳；
- **只靠红绿区分**：常见色觉差异下容易失效；
- **深底白字用于所有框**：每个模块都像重点，无法建立层级；
- **不同图重复发明色板**：读者必须重新学习方法与颜色的对应关系；
- **用颜色补救布局**：阅读顺序有问题时，再多颜色也无法修复结构。
