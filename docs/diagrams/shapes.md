# 12 形状与连线：建立视觉语法

形状不是装饰。它应该像代码中的类型一样，告诉读者“这是什么”；箭头则像操作符，告诉读者“它们如何发生关系”。稳定的形状语义能显著降低总览图的理解成本。

::: info NOTE
**本章导学**

- 为数据、模块、状态、输出和分组建立固定形状
- 区分信息流、控制关系、可选路径与反馈路径
- 控制圆角、描边、图标和三维效果，避免视觉风格混杂
- 用重复组件表达结构，而不是逐个手工重画
:::

## 一套够用的形状词典

论文示意图不需要展示 PPT 形状库的丰富程度。五到七种基础形式已经足够覆盖大多数机器学习流程。

| 对象 | 推荐形状 | 视觉规则 |
| --- | --- | --- |
| 输入数据、张量、特征图 | 直角矩形 | 细描边；需要时标尺寸，不使用厚重阴影 |
| 模型、算法、处理步骤 | 轻微圆角矩形 | 圆角保持克制，所有模块一致 |
| 状态、嵌入、潜变量 | 圆形或胶囊形 | 只在“状态”确实是独立概念时使用 |
| 数据库或持久存储 | 圆柱 | 仅表示存储，不用来表示普通数据集名称 |
| 决策或条件分支 | 菱形 | 适合算法流程，不建议滥用于模型架构 |
| 输出、结论、核心指标 | 加粗描边的矩形 | 通过边框和位置收束阅读路径 |
| 阶段、子系统、训练范围 | 无填充或浅填充容器 | 容器负责分组，不参与主流程 |

图中如果出现两个形状相同但语义完全不同的对象，或者两个语义相同但形状不同的对象，都应该重新整理。

## Oral 论文怎样建立形状词典

### CLIP：同一概念在三个阶段保持同一外形

![CLIP 论文 Figure 1，文本、图像、编码器、特征向量与相似度矩阵使用稳定的形状语义](/paper-figures/clip-overview-figure-1.png)

  <p>原图来自 <a href="https://icml.cc/virtual/2021/oral/9194"><em>Learning Transferable Visual Models From Natural Language Supervision</em></a>。上方为论文原图，不是教程自制示意图。</p>

CLIP 使用的形状并不多，但每一种都很稳定：叠放卡片表示一批样本，梯形表示 encoder，窄矩形表示单个 embedding，网格表示两两相似度。右侧进入零样本预测后，同样的梯形和窄矩形再次出现，读者无需图例就能确认对象身份。

这给 PPT 绘图一个直接规则：先做四个母版对象，再复制，不要边画边决定。

| 母版对象 | 图中语义 | PPT 制作要点 |
| --- | --- | --- |
| 叠放卡片 | 一批图片或文本 | 复制 2–3 层并轻微错位，最上层保留内容 |
| 梯形 | 编码器 | 左右面一致，文字居中，不增加三维阴影 |
| 窄矩形 | 特征向量 | 同宽同高，颜色跟随模态 |
| 网格 | 两两匹配或分数 | 用表格或等宽矩形阵列，突出少数关键格 |

### Glow-TTS：镜像布局让连线差异自己显现

![Glow-TTS 论文 Figure 1，训练与推理使用相同模块形状和相反方向的部分连线](/paper-figures/glow-tts-training-inference-figure-1.png)

<div class="paper-caption">
  <strong>NeurIPS 2020 Oral · Glow-TTS Figure 1</strong>
  <p>原图来自 <a href="https://proceedings.neurips.cc/paper/2020/hash/5c3b99e8f92532e5ad1556e53ceea00c-Abstract.html"><em>Glow-TTS</em></a>。原始图注为 “Training and inference procedures of Glow-TTS.”</p>
</div>

这张图把 Encoder、Project、Duration Predictor、Decoder 都画成克制的矩形，只把 alignment 放进更大的圆角容器。外层容器不是“另一个模型模块”，而是一个需要展开阅读的机制区域。训练与推理两栏使用相同组件尺寸，变化主要由箭头方向、`stop_gradient` 和内部 alignment 表达。

因此，画对比流程时应先复制整个共享骨架，再修改差异。左右两边若从零分别绘制，即使逻辑正确，也很容易出现模块宽度、箭头角度和间距不一致，削弱比较。

## 圆角不是越大越现代

科研图使用轻微圆角可以区分“处理模块”和“数据对象”，但胶囊形会占用更多水平空间，也容易让图看起来像产品界面。

一个稳定规则是：

- 数据：直角；
- 计算模块：2–4 px 视觉圆角，或在 PPT 中约 `0.1–0.2 cm`；
- 状态：完全圆形或胶囊形；
- 阶段容器：直角或极小圆角。

只有核心模块可以通过更粗描边或浅色填充提升权重，不要再额外改变圆角大小。

## 箭头表达关系，而不是填空

| 关系 | 推荐连线 | 示例 |
| --- | --- | --- |
| 主信息流 | 1–1.25 pt 实线箭头 | 输入进入编码器，再进入分类器 |
| 次要或可选路径 | 0.75–1 pt 虚线箭头 | 训练时才使用的辅助监督 |
| 反馈与循环 | 细曲线箭头 | 迭代更新、闭环控制 |
| 对应、共享、约束 | 无箭头细线或括号 | 权重共享、成对样本 |
| 分组归属 | 容器，不使用连线 | 多个模块属于同一阶段 |

箭头旁的文字写动作，例如 `encode`、`align`、`update`；形状内写对象或模块，例如 `Image encoder`、`Shared embedding`。这样读者会自然读成一句话。

::: warning 避免线条交叉
交叉线不是“复杂模型不可避免”，通常是分组或阅读方向还没有理顺。优先移动模块、增加中间汇合点，或者把次要关系移到独立小面板。
:::

## 连接符必须吸附到形状

在 PPT 中使用“连接符”，让端点吸附在形状连接点。普通直线在模块移动后不会跟随，很容易出现悬空、穿过边框或箭头长度不一致。

主流程尽量只选择一个方向：

- 横向图统一从左到右；
- 纵向图统一从上到下；
- 反馈路径可以反向，但必须贴近外侧边缘；
- 不要让局部流程同时使用左右、上下和对角线三种方向。

## 用容器建立层级

容器适合表达“这些对象属于同一个阶段”，例如 `Pre-training`、`Fine-tuning`、`Inference`。容器应当比内部元素更安静：

- 边框使用浅灰或细虚线；
- 标题放在左上角，字号与二级标签一致；
- 内边距四周一致；
- 不使用强饱和填充；
- 容器之间保留比内部模块更大的间距。

不要把容器画成一个新的大模块，再让主箭头同时穿过容器和内部对象。它的职责是分组，而不是再次表达计算。

## 图标只在识别更快时使用

相机、文本、音频波形、用户、服务器等熟悉图标可以缩短标签，但抽象模型模块不应强行图标化。

使用图标时：

- 选择同一套线性或面性风格；
- 统一描边粗细与视觉尺寸；
- 图标旁保留文字，不让图标单独承担专业语义；
- 避免从不同网站混用透视、色彩和圆角风格不同的素材；
- 不用装饰性插画替代真实输入或输出样例。

## 神经网络模块的表达

网络结构最容易陷入“每一层都画出来”。先判断这一层是否影响论文论点。

| 层级 | 应展示什么 | 适合放在哪里 |
| --- | --- | --- |
| 总览层 | 输入、关键阶段、创新模块、输出 | Figure 1 或方法总览 |
| 模块层 | 核心模块内部的数据变化 | 方法章节主图或子面板 |
| 实现层 | 层数、维度、核大小、激活函数 | 图旁标注、表格或附录 |

重复结构可以画一个完整单元，再用 `×N` 或括号说明次数。只有当不同层具有不同作用时，才逐层展开。

尺寸标注使用统一顺序，例如 `T × D`、`H × W × C`，并在图注中解释符号。不要有的模块写通道数，有的模块写完整张量形状，有的什么都不写。

## 形状样式参数

双栏论文图可以从下面这组规格开始：

```text
普通描边      0.75 pt
主路径描边    1.00–1.25 pt
核心模块描边  1.25–1.50 pt
辅助虚线      0.75 pt, dash 3:2
箭头头部      小型实心三角
模块圆角      0.10–0.20 cm
模块内边距    0.20–0.30 cm
```

线条太细会在 PDF 缩放或打印中消失；线条太粗则会让每个框都显得笨重。导出后必须在最终尺寸检查。

## 常见失败模式

- 立方体、圆柱、云朵和胶囊形混在一起，却没有语义差异；
- 所有模块都有阴影、渐变和高光，像流程模板而不像论文；
- 箭头宽度、箭头头部和虚线节奏不统一；
- 一条线穿过多个模块，读者无法判断它连接谁；
- 用图标替代专业术语，结果只有作者自己知道图标含义；
- 复制模块后逐个修改，最终出现难以察觉的尺寸偏差。
